🔥 Key Troubleshooting & Challenges

🏗️ Infra Availability
Challenge
Docker 강제 종료 시 HikariCP Graceful Shutdown 실패로 좀비 커넥션 잔류 → DB 한도 초과 장애.
Solution
종료 시그널(SIGTERM) 핸들링 로직 추가 및 HikariCP, Supabase Pooler 설정 최적화로 커넥션 누수 0건 달성.
+ Terraform IaC 도입으로 프리티어 만료 시 재구축 시간 90% 단축.
🧩 Data Integrity
Challenge
기초교양 과목의 줄임말(예: 독문, 불문)이 실제 학과명과 달라 매핑 데이터 정합성 실패.
Solution
Department Mapping Service 구축. 별도 매핑 테이블을 통해 수동 작업 없이 100% 자동 매핑 및 배정 정확도 향상.
⚡ Performance
Challenge
실사용 중 필터링 검색 지연 발생 & 로그인 로직에서 JPA 엔티티 노출로 인한 런타임 에러.
Solution
1. 실행 계획 분석 후 복합 인덱스(Composite Index) 적용 (7.3배 검색 최적화).
2. DTO 패턴 전면 교체로 계층 간 결합도 감소 및 API 안정성 확보.
🖱️ UX & Concurrency
Challenge
위시리스트 더블 클릭 시 중복 데이터 삽입 오류 발생.
Solution
프론트엔드/백엔드 양측에 Debounce & Throttling (0.5s) 적용. 서버 재시작 없이 즉시 이슈 대응하여 사용자 경험 보호.

Engineering Deep Dive

🧠 AI Pipeline: "Unstructured to Structured"

수만 건의 강의 데이터 중 "복잡한 시간표 패턴"을 정규식만으로 처리하는 것은 불가능했습니다.
Gemini 2.0에게 Edge Case(A/B교시, 요일 혼합)를 명확히 학습시켜 파싱 정확도 99.9%를 달성했습니다.

""" 다음은 대학교 종합강의시간표 Excel 데이터입니다. 이를 정확하게 JSON 형식으로 파싱해주세요. **중요한 파싱 규칙:** 1. 시간표(교시) 형식이 매우 복잡합니다: - [15-116:화(7),금(7B-8)] → 여러 요일, 각각 다른 시간 - [15-403:월(7)(8)(9)] → 연속 교시 (괄호만 사용) - [15-201:월(5B-6)] → A, B 접미사 유지 중요 2. 강의실 정보는 무시하고, 요일과 교시만 추출하여 timeString에 저장 예: [15-116:화(7)] [15-317:월(3)] → "화 7 월 3" 응답은 다음과 같은 JSON 배열 형태로만 주세요: [ { "subjectName": "영작문(1)", "timeString": "화 7 월 3", "isNight": false, ... } ] """

🛠️ Connection Pool Tuning

동시 접속자 200명 발생 시 DB 커넥션 고갈(Connection Starvation)로 인한 장애 발생.
Grafana 모니터링 분석 후 HikariCP 설정을 최적화하여 해결했습니다.

# application.yml (Optimization) spring: datasource: hikari: - maximum-pool-size: 5 # Default (Too small) + maximum-pool-size: 30 # Calculated based on active threads + connection-timeout: 20000 + max-lifetime: 600000 # Prevent stale connections
